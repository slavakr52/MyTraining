Добавленные проверки:
 1. Файл не открывается повторно, если он уже открыт. Без данной проверки при повторном выборе
 пункта меню "открыть файл" в рабочий список попадала бы вся информация из файла поверх уже добавленной.
 Выбрав 10 раз, получили бы 10 копий телефонной книги
 2. Добавлена проверка при сохранении файла. Без данной проверки при выборе пункта "сохранить файл"
 (при условии, что предварительно файл не был открыт) в файл телефонной книги записывался бы пустой
 список, т.е. вся информация была бы утеряна (при сохранении используется параметр 'r')
 3. Проверка при выводе всех контактов. Если рабочий список пуст, будет выведено соответствующее
 сообщение
 4. Проверка при добавлении контакта. Мы не можем проверять по пустому списку, как при открытии файла,
 например, ибо на то она и телефонная книга, что изначально она может быть пустой и туда надо записать
 новый контакт. Но мы должны быть уверены, что файл просто пуст, почему:
    - Если файл не был открыт, и мы создадим контакт, а потом сохраним, вместо всей телефонной книги
    останется один новый контакт, остальное будет стёрто
 Поэтому на всякий случай, если у нас список пустой, сначала открываем файл (сообщаем об этом юзеру),
 потом уже создаем новый контакт. Если файл был открыт ранее и просто был пустым, ничего страшного
 не случится от слова совсем, а вот если файл забыли открыть - мы спасаем инфу
